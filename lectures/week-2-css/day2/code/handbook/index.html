Doctypes

Doctypes appear at the very top of the page. They tell the browser how the html on your page should be interpreted.

xhtml 1.1 doctype

xhtml 1.1 is a form of xml and the strictest doctype. Forcing xml rules on the document. The browser doesn't care though.

<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">

html5 doctype

The html5 doctype allows code to be written far more loosely.

<!DOCTYPE html>
<html>

Block elements

Block elements provide the main structure of the page. They consume the entire width available to them.

Headings

We represent headings with the <h1> element. The number represents the headings importance. 1 is the most important and 6 the least.

Divisions

The div element is used for parts of the document that can't be semantically described by other html block elements. It is usually used in combination with a class or id attribute.

<div class="article">
   Some very exciting news.
</div>

Paragraphs

The paragraph element encloses paragraphs.

<p>
   The cat sat on the mat.
</p>

Tables

A table in html contains 1 or more rows and 1 or more columns. In xhtml the rows should be inside a tbody element. Traditionally tables were used for formatting the page. They should not be used in this way. Tables should only be used where you want to represent tabular data.

Code Visualization
<table>
  <tbody>
    <tr>
      <th>Head col1</th>
      <th>Head col2</th>
    </tr>
    <tr>
      <td>row2 col1</td>
      <td>row2 col2</td>
    </tr>
  </tbody>
</table>
Head col1 Head col2
row2 col1 row2 col2

Unordered Lists

A collection of data that has no specific order for example, facts, menu items, shopping items etc.

  • It's cold in Sweden in the winter
  • It rains in England in the winter

Ordered Lists

A collection of data that has a specific order for example, tasks, rankings

  1. Ferrari
  2. Volvo
  3. Ford

Inline elements

An inline element does not break the flow of the page.

Span

The span element is the inline version of a div. It can be used to mark up data that can not be represented by other inline elements.

Example: I read the art of computer programming
